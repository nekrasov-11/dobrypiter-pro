# SEO-доработки главной — 2026-05-14

Итерация фокусированных правок главной страницы для индексации и ранжирования. Каждый блок — отдельный коммит, чтобы можно было ревьюить независимо.

## Что сделано

### 1. Расширен Schema.org JSON-LD `SportsActivityLocation`

Файл: `web/index.html` (блок `<script type="application/ld+json">` в `<head>`)

Добавлены поля:
- `alternateName` — «Зал бокса «Добрый Питер»» (как зовут в Яндекс.Картах)
- `description` — расширено, упомянуты возрастные группы, женская группа, тренер с 2001 года
- `image` — теперь массив из 3 URL (лого, фото зала, фото тренера)
- `priceRange` — `"₽₽"` (средний ценовой сегмент)
- `sport` — `"Бокс"`
- `areaServed` — Санкт-Петербург (City)
- `sameAs` — добавлена ссылка на 2ГИС-карточку (`70000001065295093`)
- `geo` — координаты уточнены до `59.972397, 30.381931` (точная привязка к Яндекс.Карте)

Сохранено: `telephone`, `address`, `openingHoursSpecification`, `aggregateRating` (5.0 / 78 отзывов — реальное живое число с Яндекс.Карт).

### 2. Alt-тексты и lazy loading

Файл: `web/index.html`

Заполнены описательные `alt` у всех картинок на главной:
- 6 фото в галерее «О клубе» — каждое описывает что происходит (тренер секундирует, ребёнок с медалью, спортсмен перед выходом и т. д.)
- `logo.png` — «Логотип боксёрского клуба «Добрый Питер»»
- `trener.jpg` — «Тренер по боксу Некрасов Валерий Владимирович — клуб «Добрый Питер»»

`loading="lazy"` на всех картинках ниже первого экрана уже стояло, проверено.

### 3. Расписание захардкожено статичным HTML

Файлы: `web/index.html`, `web/script.js`

Раньше в `#schedule-grid` лежал только текст «Загрузка расписания…», а JS заменял его карточками после `fetch /api/schedule`. Краулеры не видели реального расписания (Googlebot/YandexBot выполняют JS, но индексация JS-рендера медленнее и менее надёжна).

Теперь:
- В исходном HTML сразу прорисованы 5 карточек (`<article class="schedule-card">`) с реальными группами и временами — копия текущих данных из API.
- JS на загрузке всё ещё дергает `/api/schedule` и перезаписывает грид свежими данными, если API ответил. То есть админка по-прежнему работает — поправил расписание в `/admin/`, на сайте появилось.
- Если API недоступен, статика остаётся (раньше JS затирал её сообщением «Не удалось загрузить расписание»).

**Что нужно делать после правок расписания через админку.** Динамика работает: пользователь видит свежее. Но крауллер видит то, что захардкожено в HTML. Чтобы и для SEO было свежее — нужно периодически синхронизировать (вручную или хуком). См. список текущих значений: [PLACEHOLDERS-SCHEDULE.md](PLACEHOLDERS-SCHEDULE.md).

### 4. «Как добраться» — без изменений

В этой итерации блок остался как есть (адрес, метро, часы, виджет Яндекс.Карты). Раскрытие маршрутами от трёх станций метро и подробным текстом про парковку отложено.

## Что проверить

### Валидность Schema.org

1. Открыть https://validator.schema.org/
2. Вставить URL `https://dobrypiter.pro/`
3. Жмём «Run test»
4. Должно показать `SportsActivityLocation` с зелёным значком. Никаких errors.
5. Warnings про необязательные поля (`address.addressLocality` и т. п.) допустимы — у нас они заполнены, но Google может потребовать `LocalBusiness` тип для расширенных карточек. Если будут warnings — глянем отдельно.

### Alt-тексты

Открыть https://dobrypiter.pro/ → DevTools → Elements → поиск `<img` — у каждого должен быть осмысленный `alt`.

### Расписание

1. Открыть `view-source:https://dobrypiter.pro/` → найти секцию `id="schedule"`. Должны быть видны 5 `<article class="schedule-card">` с реальными данными — это и есть то, что увидит крауллер.
2. В обычном просмотре — карточки выглядят так же, как раньше (JS отрендерил их же).
3. После правки расписания в `/admin/` — публичная страница должна обновиться (JS подтягивает). Статичный HTML при этом отстанет — не страшно, пользователь видит свежее.

## Переиндексация

После деплоя на прод стоит попросить поисковики переобойти страницу.

**Яндекс.Вебмастер:**
1. Открыть https://webmaster.yandex.ru/
2. Выбрать сайт `dobrypiter.pro`
3. Слева в меню: «Индексирование» → «Переобход страниц»
4. В поле URL вписать `https://dobrypiter.pro/` → нажать «Отправить»
5. Лимит — около 10 переобходов в день, главную хватит одной отправки

**Google Search Console:**
1. Открыть https://search.google.com/search-console
2. Выбрать property `dobrypiter.pro`
3. Сверху в поле «URL inspection» вставить `https://dobrypiter.pro/` → Enter
4. После загрузки данных — «Request indexing» (запросить индексирование)
