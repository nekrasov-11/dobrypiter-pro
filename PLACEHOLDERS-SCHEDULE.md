# Расписание в HTML — что и где менять

Расписание главной страницы захардкожено как статичный HTML в `web/index.html` (секция `<section id="schedule">`) в виде пяти `<article class="schedule-card">`. JS подтягивает свежее из `/api/schedule` при загрузке и перерисовывает грид — но если редактор расписания недоступен, нужно поправить и HTML тоже.

## Текущие значения (на момент коммита)

| Группа | Дни | Время |
|---|---|---|
| Детская группа | Понедельник, Среда, Пятница | 16:00 – 17:15 |
| Подростковая группа | Понедельник, Среда, Пятница | 17:30 – 18:45 |
| Взрослая группа | Понедельник, Среда, Пятница | 20:30 – 22:00 |
| Женская группа | Вторник, Четверг | 09:30 – 10:30 |
| Женская группа | Суббота | 11:00 – 12:00 |
| Индивидуальные | Понедельник, Среда, Пятница | 10:30 – 13:30 |
| Индивидуальные | Вторник, Четверг | 07:00 – 13:00 |
| Индивидуальные | Суббота | 07:00 – 14:00 |

Это копия того, что отдаёт `https://dobrypiter.pro/api/schedule` на момент SEO-фикса. Если поменяли расписание через `/admin/`, нужно вручную обновить HTML, чтобы крауллеры тоже увидели свежие данные.

## Как менять

1. Открыть `web/index.html`, найти секцию `<section id="schedule">`.
2. Внутри — пять `<article class="schedule-card">`. У каждой — `<h3>` с названием группы и `<table class="schedule-table">` с `<tr><td>день</td><td>время</td></tr>`.
3. Поправить ячейки. Полные русские названия дней недели (`Понедельник`, `Вторник`, …) и время в формате `HH:MM – HH:MM`.
4. Закоммитить, запушить, накатить на прод.

После пуша запросить переобход страницы в Яндекс.Вебмастере (см. [SEO-CHANGES.md](SEO-CHANGES.md)).

## Будущее улучшение

Можно автоматизировать синхронизацию. Например, при `PUT /api/schedule` сервер дополнительно перерисовывает HTML-фрагмент в `web/index.html` между маркерами `<!-- AUTO_SCHEDULE_START -->` / `<!-- AUTO_SCHEDULE_END -->`. Это уже было реализовано в одной из веток (см. историю), но было упрощено при чистке. Если расписание начнёт часто меняться — стоит вернуть.
